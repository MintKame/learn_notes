# 自动配置

+ 前置知识，见_2_autoconfig
  + 其中：
  + _1_bean    ：注册组件
  + _2_property：配置绑定（根据配置文件，为组件设置属性）

## @SpringBootApplication 复合注解

1. @SpringBootConfiguration

    + @Configuration：当前是配置类

2. @ComponentScan：指定在哪些包，扫描组件

3. @EnableAutoConfiguration

    + @AutoConfigurationPackage：指定默认的包规则

        + @Import(AutoConfigurationPackages.Registrar.class)
          + Registrar的方法 `registerBeanDefinitions`
          + `register(registry, new PackageImports(metadata).getPackageNames().toArray(new String[0]))`
          
          metadata：被注解标识的类(MainApplication)，导入该类所在包的所有组件 
            
    + @Import(AutoConfigurationImportSelector.class)  容器中   批量导入   自动配置的组件

        + 调用关系：

            + getAutoConfigurationEntry(annotationMetadata)

            + getCandidateConfigurations(annotationMetadata, attributes)

            + loadSpringFactories(@Nullable ClassLoader classLoader)

              默认扫描：所有依赖下的META-INF/spring.factories
 
## 自动配置类 (XxxAutoConfiguration)

1. 自动配置类 加载所有 

    + spring-boot-autoconfigure.jar 也有META-INF/spring.factories

    + 含：所有场景（127个） 自动配置类（XxxAutoConfiguration）

      （注意：只是自动配置类，而不是场景的bean）

2. 自动配置类 按需注册（到IOC容器）

    + @Conditional 需满足条件：导入相关依赖 等

 ```java
// 需 使用原生架构，而不是响应式
@ConditionalOnWebApplication(type = ConditionalOnWebApplication.Type.SERVLET) 
// 需含 CharacterEncodingFilter 类（定义于依赖 spring-boot-starter-web） 
@ConditionalOnClass(CharacterEncodingFilter.class)
@ConditionalOnProperty(prefix = "server.servlet.encoding", value = "enabled", matchIfMissing = true)
public class HttpEncodingAutoConfiguration {}
 ```

3. 自动配置类 注入属性

    + 从 XxxProperties类 获取（初始化设置了默认值）

    + XxxProperties类 **绑定** application.properties 中的前缀 （可在配置文件设置为其他值）

```java
@EnableConfigurationProperties(ServerProperties.class) 
// 开启配置绑定功能 && 把组件自动注册到容器
public class HttpEncodingAutoConfiguration {

@ConfigurationProperties(prefix = "server", ignoreUnknownFields = true)
public class ServerProperties {}
```

4. 自动配置类 生效后：类的方法，给容器装配很多组件，提供该场景下的功能
   + 优先：用户自己配置的 bean

```java
public class HttpEncodingAutoConfiguration { 
    @Bean
	@ConditionalOnMissingBean // 优先：用户自己配置的 bean
	public CharacterEncodingFilter characterEncodingFilter()  
       
    // ==========================================================
    // 作用：防止 用户配置的 文件上传解析器（multipartResolver） id不合规范 
    @Bean
    @ConditionalOnBean(MultipartResolver.class)//容器中有这个类型组件
    // 容器中没有这个名字 multipartResolver 的组件
    @ConditionalOnMissingBean(name = DispatcherServlet.MULTIPART_RESOLVER_BEAN_NAME)
    public MultipartResolver multipartResolver(MultipartResolver resolver) { 
    return resolver;
```

+ 定制化配置（2种方法）

    + application.properties 设置（使用底层提供的bean，但更改了XxxProperties 设置的默认值）

      how：先通过XxxProperties，得知在配置文件中的前缀。
  
    + 自己 @Bean 注册组件（替换底层提供的bean）
